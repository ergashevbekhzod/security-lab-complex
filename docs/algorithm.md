# Алгоритм работы проекта

1. Пользователь скачивает проект из GitHub.
2. Пользователь запускает основной файл `start.sh`.
3. Программа проверяет наличие прав администратора.
4. Открывается интерактивная терминальная панель.
5. Пользователь выбирает нужный модуль настройки.
6. Перед изменением важных файлов создаётся резервная копия.
7. Модуль выполняет настройку выбранного компонента системы.
8. Все действия записываются в `logs/actions.log`.
9. По запросу пользователя создаётся отчёт в папке `reports/`.

## Общая схема модулей

- `system_check.sh` — анализ ОС и базовых утилит;
- `users.sh` — проверка пользователей;
- `permissions.sh` — проверка прав доступа;
- `ssh.sh` — настройка SSH;
- `firewall.sh` — настройка UFW;
- `fail2ban.sh` — установка и настройка Fail2Ban;
- `logging.sh` — проверка системных журналов;
- `report.sh` — формирование отчёта.
